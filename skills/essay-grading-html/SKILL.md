# 作文批改与HTML汇总（逐句版）

当用户要求“作文批改、批量批改、语文作文评分、输出批改报告、生成HTML展示”时使用本技能。

核心目标：

1. 先审题形成评分标准；
2. 按作文数量并行委派子代理逐篇批改；
3. 为每篇作文生成一个**独立详细 HTML 文档**（必须）；
4. 生成一个总览 HTML（班级汇总页，含跳转链接）。

## 适用场景

- 单题多篇作文批改（一个题目 + 多篇学生作文）
- 用户要求“先审题，再批改”
- 用户要求可展示/可分享/可打印的 HTML 结果
- 用户要求逐句点评、逐句润色

## 输入要求

至少需要以下信息（缺失时先向用户确认）：

- 作文题目文件路径（如 `.../essay-topic.md`）
- 学生作文目录或文件列表（如 `.../students/*.md`）
- 输出目录（默认 `outputs/essay-grading/`）

## 强制工作流（必须按顺序执行）

### 第1步：读取题目与作文清单

1. 用 `read_file` 读取题目全文。
2. 用 `glob` 或 `list_dir` 获取全部作文文件。
3. 统计作文数量 `N`，记录题目全文、作文路径列表、作文总数。
4. 若 `N=0`，停止并提示用户补充作文。

### 第2步：先委派一个“审题子代理”（禁止跳过）

必须先调用一次 `Task` 完成审题。

- `description` 建议：`审题与评分标准`
- `subagent_type` 建议：`general`
- `prompt` 必须包含题目全文与评分标准结构要求

审题子代理输出至少包含：

- 立意与切题要求
- 评分维度与权重（总分 100）
- 档位描述（优秀/中等/待改进）
- 常见失分点与扣分规则
- 本题专属关键词与偏题警示

### 第3步：按作文数量并行委派多个“批改子代理”

根据 `N`，在同一轮发起 `N` 个 `Task` 并行批改（每篇一代理）。

每个批改子代理输入必须含：

- 审题评分标准对象
- 该篇作文全文
- 作文文件路径与学生标识

每个批改子代理输出必须是统一结构（见“单篇批改结果结构”），并且必须完成：

- 分项评分与总评
- 问题定位与修改建议
- **逐句批改（覆盖全文句子）**
- **逐句润色（每句提供润色版）**
- 整篇润色稿

约束：

- 禁止串行逐篇批改，必须并发。
- 子代理只负责单篇，不做全局汇总。

### 第4步：生成HTML产物（必须是“1总览 + N单篇”）

主代理汇总后，必须写入：

1. 总览页（必需）：
   - `outputs/essay-grading/index.html`
2. 每篇独立页（必需）：
   - `outputs/essay-grading/essays/<slug>.html`（共 `N` 个）

其中 `<slug>` 推荐由文件名或学生名归一化生成（如 `zuowen-yangli-01`）。

#### 总览页 `index.html` 必须包含

- 题目、时间、作文总数、平均分、最高分、最低分
- 评分标准区（维度、权重、说明）
- 班级成绩总览表（降序）
- 每位学生的“查看详细批改”链接（跳到对应单篇 HTML）
- 1-3 条共性教学建议

#### 单篇页 `<slug>.html` 必须包含

- 学生信息、作文标题、总分、分项得分
- 原文与整体评语
- 优点、问题、可执行修改建议
- **逐句批改表**
- **逐句润色表**
- 整篇润色稿（可直接朗读/提交的改写版本）

### 第5步：对用户返回结果

最终回复必须给出：

- 总览页路径
- 全部单篇页路径列表（至少展示前 3 个，剩余可汇总说明）
- 批改作文数量、均分、分数区间
- 1-3 条共性教学建议

## 评分标准结构（建议）

```json
{
  "topic_summary": "题意拆解",
  "dimensions": [
    {"name": "立意与切题", "weight": 30, "criteria": "..."},
    {"name": "结构与层次", "weight": 20, "criteria": "..."},
    {"name": "论证与素材", "weight": 20, "criteria": "..."},
    {"name": "语言表达", "weight": 20, "criteria": "..."},
    {"name": "规范与书写", "weight": 10, "criteria": "..."}
  ],
  "deduction_rules": [
    "偏题最多扣 30 分",
    "结构严重混乱扣 10-20 分"
  ],
  "topic_warnings": ["空泛抒情", "素材堆砌但无分析"]
}
```

## 单篇批改结果结构（必须）

```json
{
  "essay_path": "students/student-01.md",
  "student_name": "张三",
  "title": "作文标题",
  "total_score": 82,
  "dimension_scores": [
    {"name": "立意与切题", "score": 25, "max": 30, "comment": "..."}
  ],
  "overall_comment": "...",
  "strengths": ["..."],
  "issues": ["..."],
  "actionable_revisions": ["..."],
  "sentence_reviews": [
    {
      "idx": 1,
      "original": "原句",
      "problem_type": "表达重复/逻辑跳跃/病句/用词不准/标点问题/无",
      "diagnosis": "问题诊断",
      "revision_suggestion": "修改建议",
      "polished": "润色后句子",
      "change_explanation": "为什么这样改"
    }
  ],
  "full_polished_essay": "整篇润色稿"
}
```

## 逐句批改与润色规则（硬约束）

- 必须按原文句序逐句处理，`idx` 连续递增。
- `sentence_reviews` 的条数应覆盖全文句子，禁止只挑几句。
- 对“无明显问题”的句子也要给出 `polished`（可微调并说明）。
- 每句都要有“问题诊断 + 修改建议 + 润色结果 + 改动说明”。
- 润色要保持原意，不得凭空扩写与题意无关内容。

## HTML质量要求（硬约束）

- 文档标题统一：`作文批改报告`
- 使用内联 CSS，页面简洁、层次清晰、可打印
- 兼容窄屏阅读（基础响应式）
- 单篇页中“逐句批改/逐句润色”必须用清晰表格或卡片分组展示
- 单篇页必须提供“返回总览页”链接

## 失败与回退策略

- 审题子代理失败：重试 1 次；仍失败则停止并报告。
- 某篇批改失败：继续处理其他作文；在总览页标注该篇失败原因。
- 单篇 HTML 写入失败：写入备选路径并在总览页标红提示。
- 总览页写入失败：至少返回单篇页路径与失败原因，避免静默失败。
