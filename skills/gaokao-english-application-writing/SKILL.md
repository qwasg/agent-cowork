# 高考英语应用文批改与 HTML 汇总

当用户要求“高考英语应用文批改、应用文评分、书信/通知/演讲稿/倡议书批改、批量英语应用文批改、输出 HTML 报告”时使用本技能。

核心目标：

1. 先审题形成评分标准（紧扣高考应用文评分维度）；
2. 按作文数量并行委派子代理逐篇批改；
3. 为每篇作文生成一个**独立详细 HTML 文档**（必须）；
4. 生成一个总览 HTML（班级汇总页，含跳转链接）。

## 适用场景

- 高考英语应用文（书信、邮件、通知、演讲稿、倡议书、报道等）
- 单题多篇作文批改（一个题目 + 多篇学生作文）
- 用户要求“先审题，再批改”
- 用户要求可展示/可分享/可打印的 HTML 结果
- 用户要求逐句点评、逐句润色

## 输入要求

至少需要以下信息（缺失时先向用户确认）：

- 作文题目文件路径（如 `.../topic.md`，含题干、要点、词数要求）
- 学生作文目录或文件列表（如 `.../students/*.md` 或 `*.txt`）
- 输出目录（默认 `outputs/gaokao-english-application/`）

## 强制工作流（必须按顺序执行）

### 第1步：读取题目与作文清单

1. 用 `read_file` 读取题目全文（含文体、要点、词数限制）。
2. 用 `glob` 或 `list_dir` 获取全部作文文件。
3. 统计作文数量 `N`，记录题目全文、作文路径列表、作文总数。
4. 若 `N=0`，停止并提示用户补充作文。

### 第2步：先委派一个“审题子代理”（禁止跳过）

必须先调用一次 `Task` 完成审题。

- `description` 建议：`应用文审题与评分标准`
- `subagent_type` 建议：`general`
- `prompt` 必须包含题目全文与高考应用文评分结构要求

审题子代理输出至少包含：

- 文体判定（书信/通知/演讲稿等）与格式要求
- 要点覆盖清单（题目要求的每个要点是否必须体现）
- 评分维度与权重（总分 15，默认按高考应用文惯例；若用户指定其他总分则遵从）
- 档位描述（第五档/第四档/第三档/第二档/第一档 或 优秀/良好/及格/待提高）
- 常见失分点（要点遗漏、格式错误、时态人称、中式英语、词数不达标）
- 本题专属警示（如称呼、落款、日期、标题格式）

### 第3步：按作文数量并行委派多个“批改子代理”

根据 `N`，在同一轮发起 `N` 个 `Task` 并行批改（每篇一代理）。

每个批改子代理输入必须含：

- 审题评分标准对象
- 该篇作文全文
- 作文文件路径与学生标识

每个批改子代理输出必须是统一结构（见“单篇批改结果结构”），并且必须完成：

- 分项评分与总评
- 要点覆盖检查（逐条标注已覆盖/遗漏）
- 格式与语域检查（称呼、落款、语气是否得体）
- 问题定位与修改建议
- **逐句批改（覆盖全文句子）**
- **逐句润色（每句提供润色版）**
- 整篇润色稿（符合词数要求）

约束：

- 禁止串行逐篇批改，必须并发。
- 子代理只负责单篇，不做全局汇总。

### 第4步：生成 HTML 产物（必须是“1总览 + N单篇”）

主代理汇总后，必须写入：

1. 总览页（必需）：
   - `outputs/gaokao-english-application/index.html`
2. 每篇独立页（必需）：
   - `outputs/gaokao-english-application/essays/<slug>.html`（共 `N` 个）

其中 `<slug>` 推荐由文件名或学生名归一化生成（如 `student-01`）。

#### 总览页 `index.html` 必须包含

- 题目、文体、时间、作文总数、平均分、最高分、最低分
- 评分标准区（维度、权重、档位说明）
- 班级成绩总览表（降序）
- 每位学生的“查看详细批改”链接（跳到对应单篇 HTML）
- 1-3 条共性教学建议（如时态、要点、衔接词）

#### 单篇页 `<slug>.html` 必须包含

- 学生信息、作文标题/文体、总分、分项得分
- 要点覆盖表（✓/✗）
- 原文与整体评语
- 优点、问题、可执行修改建议
- **逐句批改表**（原句 / 问题类型 / 诊断 / 建议）
- **逐句润色表**（原句 / 润色句 / 改动说明）
- 整篇润色稿（可直接提交的改写版本）
- 词数统计（原稿 vs 润色稿）

### 第5步：对用户返回结果

最终回复必须给出：

- 总览页路径
- 全部单篇页路径列表（至少展示前 3 个，剩余可汇总说明）
- 批改作文数量、均分、分数区间
- 1-3 条共性教学建议

## 评分标准结构（建议，总分 15）

```json
{
  "genre": "letter|notice|speech|...",
  "topic_summary": "题意与要点拆解",
  "required_points": ["要点1", "要点2"],
  "dimensions": [
    {"name": "内容要点", "weight": 5, "criteria": "要点齐全、内容合理"},
    {"name": "篇章结构", "weight": 3, "criteria": "层次清晰、衔接自然"},
    {"name": "语言表达", "weight": 5, "criteria": "语法词汇准确、句式多样"},
    {"name": "格式与语域", "weight": 2, "criteria": "文体格式正确、语气得体"}
  ],
  "word_limit": {"min": 80, "max": 120},
  "deduction_rules": [
    "要点遗漏每项扣 1-2 分",
    "词数不足扣 1 分",
    "格式严重错误扣 1-2 分"
  ]
}
```

## 单篇批改结果结构（必须）

```json
{
  "essay_path": "students/student-01.md",
  "student_name": "张三",
  "genre": "letter",
  "total_score": 12,
  "word_count": 98,
  "point_coverage": [
    {"point": "要点1", "covered": true, "evidence": "..."}
  ],
  "dimension_scores": [
    {"name": "内容要点", "score": 4, "max": 5, "comment": "..."}
  ],
  "overall_comment": "...",
  "strengths": ["..."],
  "issues": ["..."],
  "actionable_revisions": ["..."],
  "sentence_reviews": [
    {
      "idx": 1,
      "original": "原句",
      "problem_type": "grammar/tense/collocation/chinglish/format/none",
      "diagnosis": "问题诊断",
      "revision_suggestion": "修改建议",
      "polished": "润色后句子",
      "change_explanation": "为什么这样改"
    }
  ],
  "full_polished_essay": "整篇润色稿"
}
```

## 逐句批改与润色规则（硬约束）

- 必须按原文句序逐句处理，`idx` 连续递增。
- `sentence_reviews` 的条数应覆盖全文句子，禁止只挑几句。
- 对“无明显问题”的句子也要给出 `polished`（可微调并说明）。
- 润色要保持原意，不得凭空添加题目未要求的要点。
- 润色稿词数须符合题目限制。

## HTML 质量要求（硬约束）

- 文档标题统一：`高考英语应用文批改报告`
- 使用内联 CSS，页面简洁、层次清晰、可打印
- 兼容窄屏阅读（基础响应式）
- 单篇页中“逐句批改/逐句润色”必须用清晰表格或卡片分组展示
- 单篇页必须提供“返回总览页”链接

## 失败与回退策略

- 审题子代理失败：重试 1 次；仍失败则停止并报告。
- 某篇批改失败：继续处理其他作文；在总览页标注该篇失败原因。
- 单篇 HTML 写入失败：写入备选路径并在总览页标红提示。
- 总览页写入失败：至少返回单篇页路径与失败原因，避免静默失败。
