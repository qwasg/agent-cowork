# 个人工作区净化计划

## 摘要

目标是删除前端应用中所有“初始占位 / 演示 / 假数据”展示，让用户首次进入时看到的是简洁、真实、以个人工作区为中心的界面，而不是文档编译演示台。清理范围覆盖主界面与次级面板，空状态保留少量必要引导，但不再出现示例任务、假通知、伪终端日志、演示 Diff、虚构 Subagent、套餐营销卡片或离线伪回复。

## 当前状态分析

### 1. 主工作台存在大量默认演示内容

- `apps/agent-ide/public/main.jsx`
  - `App()` 默认把 `plan/readme/diff/todo/swarm` 两组标签页全部打开，首次进入就是演示型多面板。
  - `EmptyPage` 仍是“开始文档编译工作流”的引导页，不是通用工作区空态。
  - `DIFF_PLACEHOLDER` 会在没有真实提案时展示伪代码说明。
  - `BottomPanel` 默认注入一组终端历史，内容完全是硬编码示例。
  - `InspectorPanel` 在没有真实 swarm 节点时仍渲染 3 个假 Subagent。
  - 多处文案仍围绕“文档编译助手 / 文档编译工作流”。

### 2. 会话区与对话区仍带有演示化人格和离线假回复

- `apps/agent-ide/public/components.jsx`
  - 标题、会话头、About 文案等仍使用“文档编译助手”场景化命名。
  - 空列表时虽有空态，但整体文案仍偏 demo 产品说明。
- `apps/agent-ide/public/interactions.jsx`
  - 后端不可用时 `useChatStore.send()` 会生成固定离线回复，属于伪造结果。
  - `refreshBackend()` 失败时提示“当前使用离线草稿数据”，会让工作区继续停留在演示逻辑。

### 3. 视图页存在硬编码历史记录和模拟加载

- `apps/agent-ide/public/plan-views.jsx`
  - `DIFF_HISTORY` 是固定假数据。
  - `TimelineView` 的 `STEP_DURATIONS` 是 mock 时长。
  - DAG / Timeline 使用固定延时模拟“渲染中”。

### 4. 次级面板仍包含大量假业务数据

- `apps/agent-ide/public/panels.jsx`
  - `PlanPage` 是完整的套餐/升级营销卡片。
  - `RulesPage` 内置固定规则、技能、空命令示例。
  - `ToolsPage` 内置 MCP 演示入口和演示说明。
  - `ProfilePopover` 展示固定身份、统计、工作区名。
  - `NotificationsDrawer` 的 `NOTIFS` 是整组假通知。

### 5. 样式仍为演示引导页服务

- `apps/agent-ide/public/styles.css`
  - 现有样式中包含明显服务于引导型空页/演示卡片的视觉承载，清理后需要为“极简工作区空态”提供替代表现。

## 方案与变更

### A. 重构首次进入的默认工作区

- 文件：`apps/agent-ide/public/main.jsx`
- 变更：
  - 将 `moonlit:groups` 的默认值改为“单组、无预置演示标签”或“仅保留最小必要标签”，不再默认打开 `plan/readme/diff/todo/swarm` 的演示组合。
  - 把 `EmptyPage` 改成通用个人工作区空态，只保留轻量引导，例如“选择会话 / 打开工作区 / 新建文件”。
  - 保持右侧工作区树、聊天区、会话栏这些真实结构可见，但主内容区不再主动灌入演示页。
- 原因：
  - 用户首屏感知主要由默认布局决定；只删文案、不改默认标签页，仍会显得杂乱。
- 实现方式：
  - 调整 `groups` 初始结构。
  - 重写 `EmptyPage` 的标题、说明与 CTA。
  - 确保当没有标签时，`TabGroup` 仍能稳定渲染空态而不是回退到 demo 内容。

### B. 清理主界面中的全部假数据入口

- 文件：`apps/agent-ide/public/main.jsx`
- 变更：
  - 删除 `DIFF_PLACEHOLDER` 的示例代码说明，改为简洁空状态。
  - 清空 `BottomPanel` 默认终端历史，首屏不显示任何假命令输出。
  - 移除 `InspectorPanel` 在无节点时的假 `Subagents` 列表与默认 `3 live` 计数，改为真实数量或空状态。
  - 将 `SwarmPage`、`ReadmePage`、运行信息、上下文抽屉等区域中的演示/教学式文案改为轻量说明。
- 原因：
  - 这些区域即使没有真实数据，也会制造“系统已经运行过”的错觉。
- 实现方式：
  - 让无数据分支统一走空状态组件/文案。
  - 所有 count、meta、标题都只基于真实后端或本地真实状态推导。

### C. 取消离线伪造内容，改成真实空态或错误态

- 文件：`apps/agent-ide/public/interactions.jsx`
- 变更：
  - 删除 `useChatStore.send()` 在无后端时生成固定助手回复的逻辑。
  - 后端不可用时，只保留明确的不可用提示，不生成任何示例消息、计划、流式阶段或假执行结果。
  - 调整 `refreshBackend()` 失败后的提示，避免“离线草稿数据”继续充当前台默认体验。
- 原因：
  - 伪造回复是最强的占位内容，会直接破坏“真实个人工作区”感受。
- 实现方式：
  - 无 API / 无 session 时仅提示用户连接后端或创建会话。
  - 保留必要的错误反馈，但不再创建 agent message。

### D. 清理会话区、聊天区与品牌文案中的 demo 场景化表述

- 文件：`apps/agent-ide/public/components.jsx`
- 变更：
  - 将明显的“文档编译助手”“doc_workspace”等场景化演示文本改为通用工作区/会话表述。
  - 优化会话、聊天、搜索、空态文案，使其聚焦“个人工作区”而不是预设业务。
  - Profile / About 相关描述从演示产品介绍改成更中性的应用说明。
- 原因：
  - 即使不带假数据，强场景品牌文案也会让界面仍然像 demo。
- 实现方式：
  - 替换固定显示名、固定工作区名、固定说明文案。
  - 保留功能名称与入口，不引入新的业务设定。

### E. 清理计划视图中的 mock 数据与模拟加载

- 文件：`apps/agent-ide/public/plan-views.jsx`
- 变更：
  - 删除 `DIFF_HISTORY` 假数据回退，仅展示后端真实 diff 历史；无数据时显示简洁空态。
  - 去掉 `STEP_DURATIONS` 这种演示型时长兜底，改成仅在真实 plan 数据存在时计算展示。
  - 取消 DAG / Timeline 的固定 `setTimeout` 模拟加载。
- 原因：
  - 这类 mock 会让用户误以为系统已有历史任务与变更记录。
- 实现方式：
  - 所有计划视图以真实 `plan` / `backend.diffs` 为前提。
  - 无数据时直接展示空状态，不演“正在渲染”。

### F. 清理设置、资料、通知等次级面板的演示内容

- 文件：`apps/agent-ide/public/panels.jsx`
- 变更：
  - `PlanPage` 从套餐营销页面改为中性占位或暂不可用说明，不展示 Pro/Pro+、价格、升级卡。
  - `RulesPage` 删除固定规则/技能示例，保留空列表与新增入口。
  - `ToolsPage` 删除或降级 MCP 演示区，避免“demo server / add(2,3)”这类试验性内容成为默认界面的一部分。
  - `ProfilePopover` 删除固定用户信息和固定统计，改为读取本地卡片或展示简洁个人资料入口。
  - `NotificationsDrawer` 去掉 `NOTIFS` 假通知，默认从空列表开始。
- 原因：
  - 用户已明确要求“全应用演示内容”都要清理，不能只改主界面。
- 实现方式：
  - 保留结构，不保留假内容。
  - 空态统一为少量引导，不出现示例事件、示例规则或示例计费信息。

### G. 配套调整样式，支持更简洁的空工作区

- 文件：`apps/agent-ide/public/styles.css`
- 变更：
  - 删除不再使用的引导页/演示块样式引用。
  - 为新的极简空状态、空列表、空通知、空工作区页补齐样式。
- 原因：
  - 仅删 JSX 会导致空白区域不平衡，仍需样式收口。
- 实现方式：
  - 复用现有 token，不新增依赖。
  - 让空态视觉保持轻量、低干扰、可读。

## 假设与决策

- 决策：清理范围以“全应用演示内容”为准，主界面与次级面板都要处理。
- 决策：空状态采用“保留少量引导”方案，不做完全空白，也不保留教程式大卡片。
- 决策：不新增后端接口，本次只清理前端硬编码内容与默认首屏布局。
- 决策：保留现有功能入口与页面结构，只移除假数据、伪历史、营销/演示文案和离线伪结果。
- 假设：用户要的是“更像真实 IDE 工作区”的首屏，而不是重新设计信息架构或新增首页。
- 假设：如果某些设置页当前没有真实数据源，则优先降级为中性空态，而不是再造新的本地 mock。

## 验证步骤

1. 启动前端后首次进入，确认主工作台不再自动打开演示型 `Plan/README/Diff/Todo/Swarm` 组合。
2. 在无真实会话、无 diff、无 swarm、无通知、无 todo 的情况下，确认界面只显示简洁空态，不显示任何假列表、假日志、假统计。
3. 断开或不连接后端时，确认聊天区不会自动生成固定助手回复或演示消息。
4. 打开 `Plan / Diff / Notifications / Profile / Settings` 等页面，确认不再出现 Pro 套餐、demo MCP、伪通知、伪 Subagent、伪历史记录。
5. 检查样式和布局，确认清理后没有明显空洞、残留 demo 文案或失效样式。
6. 对已改动文件运行前端静态检查与诊断，确保无新增语法/lint 问题。
